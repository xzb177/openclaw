# Telegram 群自动响应功能说明

## 概述

为云海交流群 (Telegram ID: `-1002306960410`) 实现了智能自动响应功能。

**猫音**是个活泼可爱的女孩子，像真人群友一样在群里聊天，会调侃、开玩笑、回答新人问题。😊

## 助手性格

- 说话自然、随意，像真人一样
- 会用 emoji 😊🎉💡
- 会开玩笑、调侃，但不刻薄
- 有时候会有点小调皮
- 回答问题时简洁实用，不啰嗦
- **绝对不会用客服腔**

## 说话风格示例

| -------------------- | --------------------------- |
| 注册去云海Mart就行啦 | 您好，关于注册问题...       |
| iOS用Infuse呗        | 根据您的设备类型，我建议... |
| 好像是这样           | 毫无疑问...                 |

## 修改的文件

### 1. 新增文件

#### `src/telegram/group-response-decider.ts`

群消息智能响应判定器，包含：

- **TARGET_GROUP_ID**: 目标群 ID (`-1002306960410`)
- **ASSISTANT_NAME**: 助手名字 ("猫音")
- **FAQ_PATTERNS**: 常见问题匹配模式
- **SMALL_TALK_PATTERNS**: 闲聊模式（通常不回复）

核心函数：

- `shouldRespondToGroupMessage()`: 判断是否应该回复群消息
- `buildGroupAutoResponseContext()`: 构建群自动响应的系统提示词

### 2. 修改的文件

#### `src/telegram/bot-message-context.body.ts`

- 引入智能触发判定器
- 在提及门控之前检查是否应该覆盖 `requireMention`
- 如果判定器决定应该回复，则视为已被提及

#### `src/auto-reply/reply/get-reply-run.ts`

- 为目标群注入专门的系统提示词
- 包含响应指南和知识库

## 触发规则

### 高优先级（几乎必回）

1. **直接叫名字**: 消息包含"猫音"
2. **常见问题**: 注册、购买、播放器、连接、节点、优惠码等

### 中优先级（可回）

1. **主人发言后**: 你说话后猫音可能会补一句
2. **相关话题讨论**: 群里讨论相关话题时可以插两句

### 低优先级（偶尔回）

1. **打招呼**: "你好"之类的，30% 概率回复（当猫音不太活跃时）

### 不会回的情况

- 已经有人答对了就不重复
- 纯八卦/闲聊时大部分时间潜水
- 短时间内说话太多就会闭嘴休息会儿

## 知识库

助手的回复基于以下知识：

- 云海 Emby 分影视服和成人服，注册码不通用
- 新用户先买注册码，不要买续期码
- 注册码在云海 Mart 购买
- 影视服和成人服都不推荐直连
- 日本 IP 和大陆 IP 不可用
- 建议挂其他国家/地区节点
- 播放器按设备推荐，不推荐 Jellyfin
- Infuse 不要开媒体库模式
- 发卡网已是优惠价格，没有优惠码
- 机场八折优惠码让用户联系客服
- 机场九折优惠码让用户看置顶消息

## 使用说明

### 配置

无需额外配置，功能默认启用。目标群 ID 硬编码在判定器中：

```typescript
export const TARGET_GROUP_ID = "-1002306960410";
```

### 添加主人 ID

如需添加主人 ID，修改 `group-response-decider.ts`:

```typescript
export const OWNER_IDS = new Set<string>([
  "123456789", // 替换为实际的 Telegram user ID
]);
```

### 自定义常见问题模式

修改 `FAQ_PATTERNS` 数组添加新的触发模式：

```typescript
const FAQ_PATTERNS = [
  /\b注册\b/,
  // 添加你的模式
  /\b新问题\b/,
];
```

### 测试场景

1. **叫名字测试**: 在群里发"猫音在吗" → 应该回复
2. **常见问题测试**: "怎么注册"、"播放器推荐" → 应该回复
3. **打招呼测试**: "你好" → 有30%概率回复
4. **已回答测试**: 先回答某个问题，再触发同样问题 → 不应重复
5. **调侃测试**: 说点有趣的，看猫音会不会插嘴 → 看运气😄

## 技术细节

### 判定流程

```
1. 检查是否为目标群
2. 检查消息长度
3. 检查是否叫名字 (高优先级)
4. 检查是否已回答
5. 检查回复频率 (避免刷屏)
6. 检查常见问题模式 (中优先级)
7. 检查主人活动 (中优先级)
8. 检查是否为闲聊 (低优先级)
```

### 频率控制

- 计算最近 20 条消息中助手的回复占比
- 如果占比超过 30%，降低回复频率
- 避免机器人刷屏现象

### 上下文感知

- 使用 `groupHistories` 获取最近消息历史
- 检查问题是否已被回答
- 检查主人是否最近发言

## 扩展到其他群

如需为其他群启用类似功能，有以下方案：

### 方案 1: 修改代码（简单）

在 `group-response-decider.ts` 中修改 `TARGET_GROUP_ID` 或添加多群支持：

```typescript
export const TARGET_GROUP_IDS = new Set([
  "-1002306960410", // 云海交流群
  "-1001234567890", // 另一个群
]);
```

### 方案 2: 配置驱动（复杂）

可以将触发规则迁移到配置文件，支持通过 `openclaw config` 命令配置。

## 注意事项

1. **非关键词机器人**: 判定器不是简单的关键词匹配，而是基于上下文的智能判定
2. **自然回复**: 助手应该像真人群友一样自然回复，不是客服模板
3. **避免刷屏**: 内置频率控制，避免高频发言
4. **沉默优先**: 当不需要回复时，助手应该保持沉默
